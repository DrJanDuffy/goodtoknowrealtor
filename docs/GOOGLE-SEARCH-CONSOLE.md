# Google Search Console setup (Good To Know REALTOR®)

Use this checklist to add and maintain the site in Google Search Console.

## 1. Add the property

1. Go to [Google Search Console](https://search.google.com/search-console).
2. **Add property**: Choose **URL prefix** and enter `https://www.goodtoknowrealtor.com` (must match the canonical `SEO_CONFIG.siteUrl` in `src/lib/seo.ts`).
3. Complete verification (step 2).

## 2. Verify ownership

The site supports two methods. Use the one GSC offers.

### Option A: HTML meta tag (recommended)

1. In GSC, when asked to verify, select **HTML tag**.
2. Copy the `content` value from the tag, e.g. `abc123xyz`.
3. In **Vercel** (or your host): Project → Settings → Environment Variables.
4. Add:
   - **Name:** `GOOGLE_SITE_VERIFICATION`
   - **Value:** the code you copied (e.g. `abc123xyz`)
   - **Environment:** Production (and Preview if you want staging verified).
5. Redeploy so the root layout picks up the variable. The layout already outputs:
   ```html
   <meta name="google-site-verification" content="..." />
   ```
6. In GSC, click **Verify**.

### Option B: HTML file upload

1. In GSC, select **HTML file upload** and note the filename (e.g. `google123abc.html`).
2. Place the downloaded file in the project’s `public/` folder so it is served at `https://www.goodtoknowrealtor.com/google123abc.html`.
3. Deploy, then in GSC enter that URL and click **Verify**.

## 3. Submit the sitemap

1. In GSC: **Sitemaps** (left menu).
2. Under **Add a new sitemap**, enter: `sitemap.xml`
3. Click **Submit**.

The sitemap URL will be: `https://www.goodtoknowrealtor.com/sitemap.xml`

- Generated by `src/app/sitemap.ts`.
- Includes all main pages, area/neighborhood pages, blog posts, and blog categories.
- `robots.txt` already references this URL (see `src/app/robots.ts`).

## 4. Request indexing for key URLs (optional)

After verification:

1. Use **URL Inspection** (top search bar).
2. Enter the URL (e.g. `https://www.goodtoknowrealtor.com/`).
3. Click **Request indexing** for homepage, `/contact`, `/listings`, and a few key area pages if you want them crawled soon.

## 5. Ongoing checks

- **Performance:** Monitor queries, clicks, impressions, and average position.
- **Coverage / Pages:** Fix any reported errors (404s, redirects, etc.).
- **Core Web Vitals:** Use the report under **Experience**; the site uses Vercel Speed Insights and Web Vitals.
- **Mobile usability:** Confirm no mobile errors.
- **Structured data:** Use **Enhancements** to confirm FAQ, Review, and other schema; fix any errors (schema is in `src/lib/seo.ts` and page-level JSON-LD).

## 6. NAP and local SEO

- Keep NAP consistent with the [NAP/GBP audit](NAP-GBP-AUDIT.md).
- In GSC, if you use a **Business Profile** (GBP), link it to the same property where applicable.

## Environment variable summary

| Variable | Purpose |
|----------|---------|
| `GOOGLE_SITE_VERIFICATION` | Meta tag and HTML-file verification code for Search Console |

Canonical site URL is set in code: `src/lib/seo.ts` → `SEO_CONFIG.siteUrl` (`https://www.goodtoknowrealtor.com`). Keep this in sync with the property URL you add in GSC.
